# Security Policy

## Reporting a vulnerability

Please do not open a public GitHub issue for security problems.

Report vulnerabilities privately to:

- `contact@engram-memory.dev`

Include:

- affected version
- deployment mode or install method
- reproduction steps
- impact
- logs, screenshots, or proof-of-concept details if they help explain the issue

We will acknowledge reports as soon as reasonably possible, investigate them, and coordinate disclosure with the reporter. Please give us reasonable time to validate and ship a fix before publishing details.

## Supported versions

Security fixes are expected only for the latest released series.

| Version | Supported |
|---------|-----------|
| `0.3.x` | Yes |
| older   | No |
